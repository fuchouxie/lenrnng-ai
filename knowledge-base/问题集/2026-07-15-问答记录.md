# 2026-07-15 问答记录

> 学员在第一天学习中提出的问题和解答摘要。

---

## Q1: `pip install openai python-dotenv` 这一步在干什么

`pip` = Python 的"应用商店"。`pip install xxx` = 从网上下载别人写好的代码，让你能 `import` 使用。

- `openai`：封装和 AI 服务器的 HTTP 通信，你不用手写请求
- `python-dotenv`：从 `.env` 文件读 Key，避免 Key 硬编码在代码里泄露

---

## Q2: `openai` 包是 OpenAI 公司的吗？所有 AI 应用都需要用这个包？

**是 OpenAI 公司的**，但不是只有 OpenAI 能用。

DeepSeek 能用的原因是：**DeepSeek 主动兼容了 OpenAI 的 API 格式**（就像插座统一标准）。

各家有自己的 SDK（Anthropic 用 `anthropic`、Google 用 `google-generativeai`），但越来越多的厂商选择兼容 OpenAI 格式。所以你会用 `openai` 包能调大部分模型。

真正的 AI 产品会写 **Provider 抽象层**，底层切换模型只需改一行配置。

---

## Q3: Claude Code、Codex 的原理也是这样吗？每次把历史全发回去？

**原理一样，工程多了一个 compact（压缩）。**

- 原理：每次把整个 messages 数组发回去 → 这就是 LLM 的"记忆"
- 工程：对话太长会先让 AI 把历史总结成摘要（compaction），防止 messages 撑爆上下文窗口

---

## Q4: Token 和字符串是什么关系？

Token 是 AI 读取文字的最小单位。字符串发过去要先切分成 Token，每个 Token 对应一个数字 ID，AI 处理的是数字。

- 英文约 0.75 个单词 ≈ 1 Token
- 中文约 1.5-2 个汉字 ≈ 1 Token
- API 按 Token 数收费

---

## Q5: 上下文窗口一百万的含义就是 messages 数组长度吗？

本质上是。上下文窗口 = messages 里所有内容 + 工具定义 + AI 输出 的总 Token 数上限。
